🔲 Layer 4: Faceting

Faceting ermöglicht es, dieselbe Grafik für verschiedene Gruppen nebeneinander zu zeichnen.
So lassen sich Muster zwischen Gruppen direkt vergleichen.

ImportantDie zwei Faceting-Funktionen
  • facet_wrap(~ Variable) → Panels werden in einer flexiblen Reihe/Spalte angeordnet
  • facet_grid(Zeile ~ Spalte) → Panels werden in einem starren Raster angeordnet

facet_wrap()

ggplot(data = df |> filter(!is.na(geschlecht) & !is.na(distress)),
       aes(x = distress)) +
  geom_histogram(binwidth = 2, fill = "steelblue", color = "white") +
  facet_wrap(~ geschlecht) +
  labs(
    title = "Verteilung von Distress nach Geschlecht",
    x     = "Distress-Score",
    y     = "Häufigkeit"
  )
NoteNützliche Argumente für facet_wrap()
  • ncol = 2 → Anzahl der Spalten festlegen
  • scales = "free_x" / "free_y" / "free" → Achsen pro Panel frei skalieren
  • labeller = label_both → zeigt auch den Variablennamen im Panel-Titel

facet_grid()

facet_grid() erstellt ein zweidimensionales Raster – eine Variable für Zeilen, eine für Spalten.

df |> filter(!is.na(altersgruppe)) |>
ggplot(aes(x = distress)) +
  geom_histogram(binwidth = 2, fill = "steelblue", color = "white") +
  facet_grid(altersgruppe ~ geschlecht) +
  labs(
    title = "Distress nach Altersgruppe und Geschlecht",
    x     = "Distress-Score",
    y     = "Häufigkeit"
  )

📊 Layer 5: Statistics

Der Layer Statistics beschreibt statistische Transformationen, die ggplot2 auf die Rohdaten anwendet, bevor sie gezeichnet werden.

Viele Geome wenden bereits intern eine statistische Transformation an – oft ohne dass wir es bemerken:

NoteGeome und ihre versteckten Stats
Geom Interne Transformation
geom_histogram() stat_bin() – zählt Werte pro Klasse
geom_bar() stat_count() – zählt Häufigkeiten
geom_boxplot() stat_boxplot() – berechnet Quartile
geom_smooth() stat_smooth() – berechnet Trendlinie
geom_point() stat_identity() – keine Transformation

stat_summary() – Kennwerte direkt in der Grafik

Mit stat_summary() können wir statistische Kennwerte direkt in eine Grafik einzeichnen –
z. B. Mittelwerte, Konfidenzintervalle oder Standardfehler.

ggplot(data = df, aes(x = geschlecht, y = distress)) +
  stat_summary(fun = mean, geom = "point", size = 4, color = "darkred") +
  stat_summary(fun.data = mean_se, geom = "errorbar", width = 0.2, color = "darkred") +
  labs(
    title   = "Mittlerer Distress nach Geschlecht (mit Standardfehler)",
    x       = "Geschlecht",
    y       = "Distress-Score (Mittelwert ± SE)",
    caption = "Eigene Darstellung | Workshop-Datensatz"
  )
Notestat_summary() – Wichtige Argumente
Argument Funktion
fun Funktion für den Punktschätzer (z. B. mean, median)
fun.data Funktion für Punkt + Intervall (z. B. mean_se, mean_cl_boot)
geom Wie wird der Kennwert dargestellt? ("point", "errorbar", "bar", "line")

Boxplot + Mittelwert kombiniert

ggplot(data = df, aes(x = geschlecht, y = distress, fill = geschlecht)) +
  geom_boxplot(alpha = 0.5, outlier.shape = NA) +
  stat_summary(fun = mean, geom = "point",
               shape = 23, size = 3, fill = "white", color = "black") +
  scale_fill_brewer(palette = "Set2") +
  labs(
    title    = "Distress nach Geschlecht",
    subtitle = "Box = Median/IQR, Raute = Mittelwert",
    x = "Geschlecht", y = "Distress-Score", fill = "Geschlecht"
  )

📏 Layer 6: Coordinates

Der Layer Coordinates legt fest, in welchem Koordinatensystem die Grafik dargestellt wird
und wie die Achsen skaliert oder transformiert werden.

Standardmäßig verwendet ggplot2 das kartesische Koordinatensystem (coord_cartesian()).
Für die meisten Grafiken müssen wir diese Schicht nicht explizit setzen – aber sie bietet nützliche Werkzeuge zur Achsenanpassung.

Achsenlimits setzen – coord_cartesian()

ggplot(data = df, aes(x = distress)) +
  geom_histogram(binwidth = 2, fill = "steelblue", color = "white") +
  coord_cartesian(xlim = c(5, 25)) +
  labs(title = "Distress-Score (nur Bereich 5–25)", x = "Distress", y = "Häufigkeit")
Warningcoord_cartesian() vs. xlim() / ylim()
  • coord_cartesian(xlim = ...) → zoomt in den Bereich, alle Daten bleiben erhalten
  • xlim() / scale_x_continuous(limits = ...) → entfernt Datenpunkte außerhalb des Bereichs

👉 Für Boxplots und geom_smooth() immer coord_cartesian() bevorzugen,
damit die statistischen Berechnungen auf allen Daten basieren.

Achsen umdrehen – coord_flip()

ggplot(data = df, aes(x = bildung)) +
  geom_bar(fill = "steelblue") +
  coord_flip() +
  labs(title = "Bildungsniveau der Befragten", x = NULL, y = "Häufigkeit")
Tip

coord_flip() ist besonders praktisch bei Balkendiagrammen mit langen Kategorienamen – die Beschriftungen auf der y-Achse sind horizontal und damit leichter lesbar.

Achsentransformation – scale_x_log10()

Mit scale_x_log10() oder scale_y_log10() können Achsen logarithmisch skaliert werden – sinnvoll bei stark rechtsschiefen Verteilungen:

ggplot(data = df, aes(x = distress)) +
  geom_histogram(binwidth = 0.1, fill = "steelblue", color = "white") +
  scale_x_log10() +
  labs(
    title   = "Distress-Score (log-skaliert)",
    x       = "Distress (log10)",
    y       = "Häufigkeit",
    caption = "Hinweis: log10-Transformation der x-Achse"
  )

Tortendiagramm – coord_polar()

Tortendiagramme sind in der wissenschaftlichen Praxis wenig empfohlen – sie machen Proportionen schwerer vergleichbar als Balkendiagramme. Trotzdem kommen sie in Berichten und Präsentationen vor.

In ggplot2 entsteht ein Tortendiagramm aus einem geom_bar() mit einem Trick: coord_polar(theta = "y") „wickelt” einen gestapelten Balken zu einem Kreis.

df |>
  count(geschlecht) |>
  mutate(anteil = n / sum(n),
         label  = paste0(geschlecht, "\n", scales::percent(anteil, accuracy = 1))) |>
  ggplot(aes(x = "", y = anteil, fill = geschlecht)) +
  geom_col(color = "white", linewidth = 0.5) +
  coord_polar(theta = "y") +
  geom_text(aes(label = label),
            position = position_stack(vjust = 0.5),
            size = 3.5, color = "white", fontface = "bold") +
  scale_fill_brewer(palette = "Set2") +
  labs(
    title   = "Geschlechterverteilung der Befragten",
    fill    = "Geschlecht",
    caption = "Eigene Darstellung | Workshop-Datensatz"
  ) +
  theme_void() +
  theme(
    plot.title      = element_text(size = 13, face = "bold", hjust = 0.5),
    legend.position = "bottom"
  )
WarningWann lieber kein Tortendiagramm?
  • bei mehr als 4–5 Kategorien → Segmente werden zu schmal
  • wenn genaue Werte verglichen werden sollen → Balkendiagramm ist besser
  • bei ähnlich großen Anteilen → kaum unterscheidbar

👉 Als Faustregel: Wenn du ein Tortendiagramm machen möchtest, überleg zuerst, ob ein geom_bar(position = "fill") nicht klarer wäre.

🖌️ Layer 7: Theme & Styling

Themes steuern das Hintergrunddesign einer Grafik:
Gitterlinien, Hintergrundfarbe, Schriften, Achsenlinien.

ggplot2 bringt bereits mehrere fertige Themes mit.

TipEmpfehlung für wissenschaftliche Grafiken

theme_minimal() oder theme_bw() sind häufig die beste Wahl für Publikationen – sie sind schlicht, gut lesbar und lenken nicht vom Inhalt ab.

theme() – Eigene Anpassungen

Mit theme() kannst du einzelne Elemente gezielt anpassen:

p +
  theme_minimal() +
  theme(
    plot.title      = element_text(size = 14, face = "bold"),
    axis.title      = element_text(size = 11),
    axis.text       = element_text(size = 10),
    legend.position = "bottom"
  )
NoteWichtige theme()-Elemente
Element Funktion
plot.title Haupttitel
plot.subtitle Untertitel
axis.title Achsentitel (beide)
axis.text Achsenbeschriftungen
legend.position "right", "left", "bottom", "none"
panel.grid Gitterlinien
plot.background Hintergrund der gesamten Grafik

Diese werden mit element_text(), element_line(), element_rect() oder element_blank() konfiguriert.

Eigenes Theme definieren

In größeren Projekten lohnt es sich, ein eigenes Standard-Theme einmal zu definieren und für alle Grafiken wiederzuverwenden:

mein_theme <- theme_minimal() +
  theme(
    plot.title       = element_text(size = 14, face = "bold", color = "#1a1a2e"),
    plot.subtitle    = element_text(size = 11, color = "#555555"),
    axis.title       = element_text(size = 10, color = "#333333"),
    axis.text        = element_text(size = 9),
    legend.position  = "bottom",
    panel.grid.minor = element_blank()
  )

p + mein_theme

patchwork – Plots kombinieren

Manchmal möchte man mehrere verschiedene Grafiken nebeneinander oder untereinander in einer Abbildung zeigen – z. B. in einer Publikation oder einem Bericht.

Dafür nutzen wir das Paket patchwork.

# install.packages("patchwork")
library(patchwork)

Grundlegende Syntax

Mit patchwork werden Plots mit +, / und | kombiniert:

p1 <- ggplot(data = df, aes(x = distress)) +
  geom_histogram(binwidth = 2, fill = "steelblue", color = "white") +
  labs(title = "Verteilung Distress", x = "Distress-Score", y = "Häufigkeit") +
  mein_theme

p2 <- ggplot(data = df, aes(x = geschlecht, y = distress, fill = geschlecht)) +
  geom_boxplot() +
  scale_fill_brewer(palette = "Set2") +
  labs(title = "Distress nach Geschlecht", x = NULL, y = "Distress-Score") +
  mein_theme +
  theme(legend.position = "none")

# Nebeneinander:
p1 | p2
# Untereinander:
p1 / p2

Layouts & Anmerkungen

p3 <- ggplot(data = df, aes(x = distress, y = soz_u)) +
  geom_point(alpha = 0.3, color = "steelblue") +
  geom_smooth(method = "lm", se = FALSE, color = "darkred") +
  labs(title = "Distress & Soziale Ungleichheit", x = "Distress", y = "Soz. Ungleichheit") +
  mein_theme

# Kombiniertes Layout: oben zwei, unten einer
(p1 | p2) / p3 +
  plot_annotation(
    title   = "Abbildung 1: Überblick Distress und Soziale Ungleichheit",
    caption = "Eigene Darstellung | Workshop-Datensatz",
    theme   = theme(plot.title = element_text(size = 14, face = "bold"))
  )
Notepatchwork – Kurzreferenz
Operator Bedeutung
p1 | p2 nebeneinander
p1 / p2 untereinander
(p1 | p2) / p3 kombinierte Layouts
plot_annotation() Gesamttitel, Caption
plot_layout(widths = c(2, 1)) Verhältnis der Breiten

# Bestimmten Plot speichern:
ggsave("output/boxplot_distress.pdf", plot = p2, width = 6, height = 4)

TipEmpfehlungen für den Export
  • Für Hausarbeiten / Berichte: dpi = 300, Format .png oder .pdf
  • Für Präsentationen: dpi = 150, breiteres Format (z. B. width = 10, height = 6)
  • Für Veröffentlichungen: .pdf oder .svg – verlustfreie Vektorgrafiken

Mini-Projekt

Zum Abschluss des Workshops erstellst du eine vollständige, publikationsreife Abbildung –
von der Datenvorbereitung bis zum Export.

Zielsetzung

Am Ende dieses Mini-Projekts habt ihr:

  • mindestens drei verschiedene Grafiken erstellt
  • Aesthetics, Themes und Farbpaletten angewendet
  • die Plots mit patchwork zu einer kombinierten Abbildung zusammengeführt
  • einen Gesamttitel und Caption hinzugefügt
  • die Abbildung als Datei exportiert

Aufgabenstellung

Schritt 1: Fragestellung wählen

Wählt eine inhaltliche Frage aus dem Datensatz, z. B.:

  • Unterscheiden sich Distress oder Soziale Ungleichheit nach Bildung, Geschlecht oder Wohnort?
  • Gibt es einen Zusammenhang zwischen Alter und Distress, der sich nach Geschlecht unterscheidet?
  • Wie verteilt sich eine Variable über verschiedene Gruppen?

👉 Haltet eure Fragestellung kurz als Kommentar im Skript fest.

Schritt 2: Datensatz vorbereiten

Wählt die relevanten Variablen aus und bereitet sie auf:

  • filter() für fehlende Werte
  • Faktor-Labels setzen (falls noch nicht geschehen)
  • ggf. neue Variablen erstellen (siehe Codebook)

Schritt 3: Drei Grafiken erstellen

Erstellt mindestens drei thematisch zusammenpassende Grafiken:

  • mindestens ein Histogramm oder Boxplot
  • mindestens ein Streudiagramm (mit geom_smooth, wenn sinnvoll)
  • mindestens ein Balkendiagramm oder eine facettierte Grafik

Anforderungen an jede Grafik:

  • labs() mit Titel, Achsenbeschriftungen und Caption
  • eigenes mein_theme angewendet
  • sinnvolle Farbpalette gewählt

Schritt 4: Plots kombinieren

Kombiniert eure drei Grafiken mit patchwork:

  • wählt ein sinnvolles Layout (|, /, oder kombiniert)
  • fügt einen Gesamttitel mit plot_annotation() hinzu

Schritt 5: Exportieren

Speichert eure kombinierte Abbildung mit ggsave():

  • Format: PNG oder PDF
  • Auflösung: dpi = 300
  • Datei im Ordner output/

Checkliste

Am Ende sollten bei euch vorhanden sein:

  • drei Einzelgrafiken (als Objekte p1, p2, p3 gespeichert)
  • eine kombinierte patchwork-Abbildung mit Gesamttitel
  • eine exportierte Datei in output/
  • ein dokumentiertes Skript mit Kommentaren
TipTipps für das Mini-Projekt
  • Fangt einfach an – erst der Grundplot, dann das Styling
  • Greift auf euer mein_theme-Objekt zurück
  • Beim Export: erst ggsave() testen, dann die Auflösung hochsetzen
  • Nutzt facet_wrap() wenn ihr viele Gruppen vergleichen wollt

Recap Workshop

NoteRecap beider Tage
  • Tag 1 – Data, Aesthetics, Geometries
  • Tag 2 – Facets, Statistics, Coordinates, Theme
  • Warum Visualisierung? Anscombe-Quartett, Exploration, Kommunikation, Qualitätskontrolle
  • Was ist ggplot2? Grammar of Graphics, schichtweiser Aufbau
  • 🗂️ Data: ggplot(data = df) – Datenbasis, Pipe in ggplot
  • 🎨 Aesthetics: aes(x, y, color, fill, size, shape, alpha) – global vs. lokal, innerhalb vs. außerhalb
  • 🎨 Aesthetics vertieft: color vs. fill, alpha, size, shape
  • 📐 Geometries: geom_point(), geom_histogram(), geom_bar(), geom_boxplot(), geom_smooth(), geom_jitter()
  • Beschriftungen: labs() mit title, subtitle, x, y, caption, color, fill
  • 🔲 Facets: facet_wrap() und facet_grid() – Gruppenvergleiche durch Teilgrafiken
  • 📊 Statistics: stat_summary() – Mittelwerte und Fehlerbalken direkt in der Grafik
  • 📏 Coordinates: coord_cartesian(), coord_flip(), scale_x_log10(), coord_polar() (Tortendiagramm)
  • 🖌️ Theme: Fertige Themes, theme()-Anpassungen, eigenes Theme-Objekt, Farbpaletten
  • patchwork: |, /, plot_annotation() – mehrere Plots kombinieren
  • Export: ggsave() mit dpi, width, height
